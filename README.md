# daily-record — 日报需求记录

日报流程第一步：总结当天完成的需求，按五维度评估工时，写入 md 表格。

## 解决了什么问题

**日报写什么、工时怎么估，每天纠结半小时。** 要么忘记记录、要么拍脑袋填数字、要么 AI 辅助时间被忽略。这个技能把评估流程标准化：设计、改动、测试、返工、沟通五个维度逐一展开，人工工时和 AI 工时分开算，不准跳过、不准糊弄。

## 完整流程

### 1. 打开今天的日报文件

自动定位到 `日报-{年}-{月}月/日报需求记录-{Y}-{M}-{D}.md`，不存在则按模板创建。

### 2. 提取上下文信息

从对话和 git 记录自动提取：仓库名、需求概述、涉及模块、备注。推断不了的逐项问。

### 3. 工时评估（五维度，禁止跳过）

| 维度 | 估算内容 |
|------|---------|
| 设计调研 | 现状摸排 + 方案设计 + 依赖追溯 + 方案对齐 |
| 代码改动 | 新增代码 + 机械替换 + 差异修改 + 上下文切换 |
| 测试验证 | 操作入口数 × 5min + 异步场景数 × 12min + 异常分支 |
| 风险返工 | (前三项之和) × 广度系数 × 深度系数 + 边界遗漏 |
| 沟通协作 | 需求对齐 + 方案讨论 + 工时评估 |

人工工时 = 五维度之和。AI 工时单独算。

### 4. 等用户确认后写入 md

## 输出格式

| 序号 | 日期 | 仓库 | 需求概述 | 涉及模块 | 状态 | 人工工时(h) | AI辅助工时(h) | 备注 |
|------|------|------|---------|---------|------|-------------|---------------|------|

状态用百分比：`100%`（已完成）、`50%`（进行中）、`0%`（未开始）。

## 安装

```bash
git clone https://github.com/huzhw/daily-record-skill.git ~/.claude/skills/daily-record
```

安装后在 AI 编码助手里说「补充日报」即可触发。
